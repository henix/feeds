原文

是的，程序君已经半个月没出来冒个泡了，哪怕是花一秒钟摁个笑脸，或者传个被我家小丫头骑在身下扮演 Sven（Frozen 里的驯鹿）的照片。

有读者发了个哭脸过来，说刚把你置顶成真爱，就再没看到文章。我回了个大哭脸，说，对不起。我们的真爱经得起时间的考验么？等了许久，就当我悲哀地感慨自己被「真爱」无情地抛弃时，对方发来一段语音 —— 我愣了半天无所适从，战战兢兢打开后发现一票磁性的 radio 音传出：affirmative，roger，go go go。哈，原来这是个 CS 迷。果然是真爱。

一般而言，我长时间没发文章只有两种可能，如果不是被病毒君劫持了，那一定是忙得分身乏术。最近我是二者兼具。感冒了一周，浑身乏力，还偏偏赶上了各种各样的事情。不过这一切都已结束，我又可以活蹦乱跳了！更令人开心的是，我终于等到了回美证的批复，也就是在默默等待了一年零七个月，我可以回国了！除了探亲访友吃喝玩乐外，这次回国，我还肩负着一个重要的使命：为我所在的公司 adRise / TubiTV 在北京招募和培训工程师！

是的，我们要在北京招募工程师 —— 拿时下吹牛逼的方式说，就是我们要建一个海外的研发中心。不过牛逼暂且留给 marketing 的人去吹，咱们工程师之间还是掏心窝子说话：我们想在北京组建一个研发团队，和旧金山的兄弟团队共同打造一系列伟大的产品。至于团队建多大，天花板是什么，我目前还无法给出一个明确的答案。作为一家创业公司，我们在稳扎稳打的基础上小步快跑。这次我回去的目标是能够招募并培训几个能力出众的工程师，随着公司的成长，我们会稳健地投入更多的资源。

这并非一时的心血来潮或者单单靠我个人的推动。去年底我加入后整个管理团队就严肃地讨论了这个方案，并且在今年初开始，花费了巨大的代价（人力，金钱和时间成本），在北京设立了一个 WFOE（外商独资公司）公司（预计7月底能完成所有法律手续）。这充分表明了一个很明确的信息：和我们在美国的其他 site 一样，我们在中国国建立一个对等的团队，吸引中国的优秀工程师和我们一起大展宏图。

如果你想了解 adRise / TubiTV 在做什么，可以参考我的这篇：派拉蒙，米高梅和狮门。团队现在有三个中国工程师，如果加上华人工程师，那就是五个；再加上正在学汉语的工程师，一共是七人，中国风横扫了半支工程师团队。

介绍一下团队里的中国工程师。

Haofei —— 北邮本科，master 就读于号称美国政治家摇篮的雪城大学，毕业后即加入 adRise。在 adRise 工作的两年时光里，他神挡杀神，佛挡杀佛，硬是从小白工程师成长为横跨亚非欧，不对，横跨 web 端，OTT 端，后端等三大端的公司第一大腿 —— 公司一大半的系统，尤其是核心的内容管理系统和视频转码系统，都在他的统帅之下。我们最近 promote 了 4 个 tech leader，haofei 是 tech lead for everything —— 公司里外事不决问创始人，内事不决问他就可以了。haofei 同学的努力程度令人发指：经常在半夜十二点还给我发 slack，说在么，然后就是一大波问题和想法袭来。。。雪大的校训是：Suos Cultores Scientia Coronat，知识为追寻他的人加冕。haofei 显然从中寻求到了力量。

Zhiye —— 北邮研究生（哇，又是北邮），前美团前端，在 adRise 变身全栈，做得了 API，还能一个人独自撑起整个 web 产品。我们如今的 tubitv.com，85% 的工作都出自他之手。产品上的成就使他成长为公司的 tech lead for web。如今他孤身一人在大德州混迹于达拉斯牛仔群中，远程操控我们加州团队在 web 上的技术发展。zhiye 由于家庭的原因，住在达拉斯，但他身在德州心系旧金山。和我们在一起时，他是个 Warrior；走在达拉斯的大街上，他必须假装是 Maverick。所以当 Warrior 客场挑战 Maverick 时，zhiye 一般不敢出门。。。

Haofei 和 Zhiye 最大的特点是：self motivated。他们的脑袋里自带罗盘，装着产品的 roadmap，公司的 roadmap，以及人生的 roadmap。不用有人催，不用刻意去关注进度 —— 他们能够把事情做好，并且，时时刻刻都在自我精进。haofei 刚入公司时在 xbox 上面写 C#，在后台折腾 php，如今做得了 angular（html5 smart tv，你家的 samsung TV 里说不定还运行着一年前他写的代码），玩得转 nodejs，折腾得了 spark；zhiye 在重构我们的 web 产品时，力排众议选用他之前也并未大规模使用过的 react，在项目中愣是把 react + redux + react router 等一众坑填平，还引领整个 web 团队学会了 react。有次在我跟一个工程师 1:1 时，他自豪地说：太赞了！我能在简历里写上精通 react / redux 了！（有没有逼宫加薪的味道~）

你看，他们二位就是我们团队的缩影：我们并非由一群牛气冲天的人组成，但我们是一群积极主动，自我激励，学习学习再学习的工程师们。不断学习，不断成长是我们的文化的重要部分 —— 和其他同阶段的创业公司不同，我们在帮助员工成长方面有充沛的预算，比如各种会议（今年我们的工程师参加了 EON - elasticsearch 会议，google I/O，Droidcon - android，预计参加的有 Node Up，还有 AWS re:invent 2016），各种在线课程（如 egghead，coursera, etc.），甚至线下的课程（我们的产品经理学了 html/css/javascript），还有同事间的交流（我们 android 的工程师现在正在跟我学 javascript）。。。

聊到这里，你大概对我们的公司和文化有一些基础认知。下面是也许让你热血沸腾的事情：

我们的所有后台服务架构搭建在 AWS 之上，是 AWS 的重度使用者。基本上你听过名字的 AWS 服务，我们都在使用或者在试验阶段。

部署系统使用 ansible。我们的 playbook 已经涵盖了 80% 以上的各种系统。一条命令，数分钟内能把代码无缝部署到所有服务器上。

尽管在局部使用 optimizely 等各式 A/B testing 工具，我们后台使用了自己开发的一套基于 facebook planout 的 experiment system，全局掌控所有产品线的 experiment 需求。

API 系统的内核是我基于 restify 写的，见我之前所写的系列文章：再谈 API 的撰写 - 总览，再谈 API 的撰写 - 架构，再谈 API 的撰写 - 子系统，再谈 API 的撰写 - 契约。如今我们已经将大部分遗留的 API 移植到这个系统之上，运行稳定。只要你使用 tubitv 的任何一款产品，其背后都有这个 API 系统的支撑。我们有意在合适的时间将其开源。

至于 analytics，我们有一套自己的体系，通过 analytics API 汇总到 kenisis，然后在 redshift 中沉淀，通过 spark cluster 做各种各样的 transformation，最终在诸如 Periscope 的工具中呈现出来。

CMS 是自己开发的，涵盖一套管理我们的内容，并让我们的 TV programming team 可以调配节目的管理系统；同时还包括一套几十个步骤的，复杂的 content pipeline，把原始的视频抽取信息，转码，并传输到 CDN 上。

广告系统支持我们所有广告客户的 realtime bidding。你在 tubitv 各个产品线看的每一个广告，都离不开这个系统的辛勤劳动。

至于客户端，我们有：roku，android phone，android pad，chromecast，iphone，ipad，apple tv，sumsang tv，fire tv，xbox 等十多个客户端平台的支持。这其中，android 产品线一度在 play store 总榜六十多名，iOS 产品线在 entertainment 排位第八。

至于机会，下面几方面我认为是重要的，值得关注的点。我不是 sales，不能瞎忽悠，所以只陈列事实，请自行从字里行间发现潜台词。

我们已经是一家稳健的，收入充沛并持续快速增长的 B 轮公司。我们在 Free TV 这个市场是北美最大的玩家，business 地位相当稳固（好莱坞三大宝 Paramount，MGM，Lionsgate 是我们的合作伙伴）。

由于公司还很小，WFOE 公司的员工能够直接和总部的任何人打交道，没有复杂的官僚体系，你不必让我或者其他人成为你的传声筒。当然，作为你的直接上司，我乐于，也有义务为你扫清一切工作上的障碍，以及替你提升在总部的形象。

只要我们的业务不出现大的变故，国内的 WFOE 会持续投入。

我们同时在着手申请 L1 blanket petition。这除了说明我们对中国 site 的严肃认真，更重要的是释放出一个积极而重要的信号：我们在需要的情况下，愿意把工程师从国内以 L1A，L1B 的方式 relocate 到美国。至于 L1A，L1B 是什么，有什么作用，请看我的这篇文章：如何获得米国工作居留的机会。

整个公司依旧很小，任何一个工程师依旧有机会通过一行行指尖流淌的代码，去影响公司（同时影响自己的人生）。


我和我的同事会定期回国，和大家一起探讨技术，人生，足球，NBA，塞浦路斯（我们的 CTO 的家乡）究竟在哪等等。

我们也会定期派人来总部和热情的同事们会面并接受技术培训。也许周末我们会开车带你到一号公路拉拉风，或者去 Muir woods Park / Yosemite 洗洗肺。甚至，如果凑巧你赶上了 Warrior 主场的常规赛比赛，花上几十美金，可以和同事一起去感受一下 NBA 的氛围。我们都是 Stephen Curry 的球迷，Curry 某种程度上是我们自己的一个图腾，我们坚信：缺失的天赋，极低的起点都可以用勤奋和努力追回。

我们崇尚 getting things done，所以拥抱拿来主义，公司并不明确限制对第三方技术的使用，所以我们使用了大量的第三方服务：ELK，datadog / new relic，twilio，sentry 等等。现在这个阶段，非核心业务，凡是能用钱解决的，我们都不想浪费你的时间。

我们崇尚为工程师提供最好的硬件：mbp 15" / thinkpad（满足不喜欢 mbp 的某些 ubuntu 狂热分子），其他的根据工程师的需要配置；软件：slack，webStore / atom / sublime（自行选择），confluence，jira，google apps 等。当然，对于国内的工程师我们会配置合适的 VPN，方便你访问各种生产工具。

我们需要什么样的人？

  • 我们需要那种像 Haofei 和 Zhiye 那样，不用 manager 盯着，能够自我管理的人。作为 remote engineer，自我驱动无比重要。

  • 我们需要能够不断学习，不拿自己的过去去定义自己的未来的人。技术变化一日千里，每个人都要随时准备去接受新的事物。

  • 我们需要 rock star，那种只要存在就能带来不同，让公司，团队变得更好的人。

  • 我们需要能够畅通无阻地表达自己观点的人。你的英文口语也许不必好，但你敢于并且善于用蹩脚的英文表达自己的思路。（而且随着时间的推移，你能够提升你的英文水平）

第三点有点抽象，你可以参考这篇文章：缘分天注定？的后半部分。

回顾我自己的职业生涯，无论是写 Patent，做各种内部工具，写各种文档，搞各种和工程师文化相关的事情，这些都没有人要求我必须做。它们也不是我的本职工作。只不过，在工作的过程中，我发现，如果我做了这些事情，可以让我周围的人和我自己要么变得更好，要么工作更轻松。我创造的价值不仅仅是公司发给我的工资的那部分，还包括我觉得一切可以改进的事情。rock star 就是那种能把团队/产品扛在肩上，做出让所有人 cheer 的事情来。唯有此，你才能不断突破职业生涯的天花板。

好，具体职位：
