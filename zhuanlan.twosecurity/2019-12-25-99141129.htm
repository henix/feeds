项目实现流程：

下载【必选】

001：

下载项目，https://github.com/testerSunshine/12306 将项目下载到本地，解压好。（最好下载最新的，因为最近更新频繁）

002：

下载 python，https://www.python.org/downloads/windows/ 下载 3.6 - 3.7.4 之间的版本。

然后安装依赖：

pip3 install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt

报错：

tensorboard 1.14.0 has requirement setuptools>=41.0.0, but you'll have setuptools 40.8.0 which is incompatible.

解决方法【任选其一】：

1.更新 setuptools 版本

pip3 install --upgrade setuptools

2.强行安装

pip3 install --ignore-installed tensorflow

003：

下载图形识别模块，https://github.com/zhaipro/easy12306 下载机器学习所需模型。

配置【必选】

TickerConfig.py

根据提示，进行必要的基础配置。

配置打码服务，有服务器的建议使用自家服务器搭建，方法在下面。

不搭建就默认使用大佬们提供的就好了。

搭建验证码识别服务器【可选】

项目地址：https://github.com/YinAoXiong/12306_code_server

拉取到服务器：

git clone https://github.com/YinAoXiong/12306_code_server.git

安装依赖：

pip3 install -r requirements.txt

下载模型文件：

bash download_model.sh

运行，默认workers为1，使用80端口，可以自行修改 gunicorn.conf

gunicorn app:app -c gunicorn.conf.py

如果服务器上有 docker，可以一键部署：

docker run -d -p 8080:80 --name 12306 yinaoxiong/12306_code_server

docker-compose 部署

version: "3"

services:
  code_12306:
    image: yinaoxiong/12306_code_server
    ports:
      - 5002:80 #可以根据需要修改端口
    environment:
      - WORKERS=1 #gunicorn works 默认为1可以根据服务器配置自行调整
    restart: always

邮箱配置，如果抢到了票，会通过邮件通知。【可选】

配置 server酱 微信提醒【可选】

什么是 server酱？
是一款 程序员 和 服务器 之间的通信软件。简而言之就是从服务器推报警和日志到手机的工具。

工作原理：

通过 github 登录然后生成 KEY，KEY 与微信的 userID 映射，服务端开放一个接口，接收你的SCKEY、title、content，然后再调用微信官方接口发送客户消息。

server酱官网：http://sc.ftqq.com/3.version

操作流程：

用 github 帐号登录。

获取 SCKEY：http://sc.ftqq.com/?c=code

绑定微信推送。

扫描关注后，点击“检查结果并绑定”就可以了。

可以先测试一下，看是否绑定成功。

测试信息地址：http://sc.ftqq.com/?c=code

最后，开启通知，将秘钥填好。

配置 cookie 方式，按照要求下载符合自己 google 浏览器版本的驱动即可。【可选】

剩下的配置点，按需配置【可选】

启动【必选】

先筛选 cdn。

python3 run.py c

启动服务。

python3 run.py r

帮助命令：

python3 run.py -h

通过 docker 搭建

如果你的服务器安装了 docker 与 docker-compose，那么你可以忽略上面的所有步骤，直接按以下步骤操作，就可以开始抢票了：

前提：

  • 请确认你安装的 docker 版本为 18.09 及以上：docker -v
  • 请确认你安装的 docker-compose 版本为1.23.2及以上：docker-compose -v
  • 请根据自己需要修改好配置文件：TickerConfig.py
  • 请修改配置文件 TickerConfig.py 中的变量 AUTO_CODE_TYPE 和 HOST，AUTO_CODE_TYPE 改为 3, HOST改为 "captcha:80"（这里很重要，这是本地打码服务器的配置）

命令：

  • 开始抢票：docker-compose up --build -d
  • 停止抢票：docker-compose down
  • 查看抢票日志：docker logs --follow ticket
