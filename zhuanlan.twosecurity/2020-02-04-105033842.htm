浏览器
顾名思义，就是浏览这世间万千网站的“神器”。

在浏览器里输入我的名字，通过它，你们才能看到我。

http://example.com

别搞混了，这可不是我的名字，这是 URL。

统一资源定位符（Uniform Resource Locator，缩写为URL），又叫做网页地址，是互联网上标准的资源的地址（Address）。
一般为：
协议://主机:端口/路径

URL 包含几个部分，最前面的是 种族。

在该 URL 中，协议部分为（http://），这代表该网页使用的是 http 协议。

我的种族是 http://

当然还有别的种族，像 https:// （网络加密的超文本传输协议）file:// （本地文件传输协议，file协议主要用于访问本地计算机中的文件）ftp:// （文件传输协议）mailto: （资源为电子邮件地址，通过SMTP访问）

名字后面是门牌号，我家是 80，来找我的时候千万别走错了。

端口号
用来区分同一台服务器上不同服务的标识（基于web服务管理创建服务的时候可以指定），不同服务之间一般是不能使用相同的端口号的。
HTTP 默认端口号 80
HTTPS 默认端口号 443
FTP 默认端口号 21

其他的我们改日再聊。

告诉浏览器我的名字，浏览器就会去找我，它可不知道我住哪儿，那它是怎么找到我的呢？

对，它会去名字系统里面查我的家庭住址。

名字系统就是 域名系统（Domain Name System 缩写 DNS，Domain Name 被译为域名）是因特网的一项核心服务，它作为可以将域名和 IP 地址相互映射的一个分布式数据库，能够使人更方便的访问互联网，而不用去记住能够被机器直接读取的 IP 数串。DNS 系统是为解析域名为 IP 地址而存在的。

当然要找到我住哪儿可没那么容易，浏览器会查很多地方。

首先是浏览器缓冲，浏览器会缓存 DNS 记录一段时间，然后查系统缓存，从 Hosts 文件查找是否有该域名和对应 IP，之后查路由器缓冲，一般路由器也会缓存域名信息 ISP DNS 缓存，比如到电信的 DNS 上查找缓存如果都没有找到，则向根域名服务器查找域名对应 IP，根域名服务器把请求转发到下一级，直到找到 IP。

浏览器来访我家，找到我的房间，最终找到了我。

是怎么找到我的呢？这里先卖个关子，下次再和你们慢慢聊。

其实，我的本质就是它——index.html

HTML 中的内容会被浏览器接受后被一句句读取解析，解析到 link 标签后重新发送请求获取 CSS 或其他资源，解析到 script 标签后发送请求获取 JS，并执行代码，解析到 img 标签后发送请求获取图片资源。

最后你们就看到我了，我就长这样。

浏览器会根据 HTML 和 CSS 渲染页面到屏幕上，JS 会被执行。
