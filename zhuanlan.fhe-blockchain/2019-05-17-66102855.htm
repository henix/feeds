上期讲了在windows上如何安装NTL，收到大家的好评。NTL是密码学库，尤其是全同态加密库的基础，例如HELIB就是建立在NTL库之上。很多朋友碰到NTL安装的问题。我们把我们的经验分享给大家。NTL库运行在linux上会更好，速度也会快一些。这期给大家讲如何在Ubuntu上安装NTL。




一、先在官网上下载好VMware，安装时需要密钥，只需要在网上搜索到有效的密钥即可。下载好后点击创建新的虚拟机，如图会有新建虚拟机的向导，按照安装向导就好。但有些要注意的地方。

















这样Ubuntu便安装好了。接下来开始安装NTL。



二、现在虚拟器上下好NTL，并将文件提取到桌面新建的ntl文件夹中。提取后已经解压好文件了。



用ctrl + alt + T 打开终端，输入sudo su输入密码后，先安装一些安装NTL库所需的必要的东西。 （g++、m4、gmp）





首先安装一下g++。（输入 apt-get install build-essential）




自动安装了g++ 4.8。安装完成后g++ --version可以查g++的版本






下载m4。（通过输入sudo apt-getinstall m4）便能下m4了 。



只要在[y/n]输入y即可。






再安装GMP，先从https://gmplib.org/ 中下载gmp-6.1.2(必须下载这个最新版本，不然在安装NTL中出现问题) 在虚拟机中下载好后，提取到桌面新建的名为 gmp的文件夹中。





通过cd gmp-6.1.2中。然后输入 按顺序输入

./configure

make

make check

sudo make install


安装完成后可以输入ls /usr/local/include/

如果安装成功会跳出 gmp.h

输入 ls /usr/local/lib/


会出现 libgmp.a libgmp.la libgmp.so libgmp.so.10 libgmp.so.10.3.0


接下来开始安装NTL库。通过cd 进入ntl文件夹的ntl-11.3.2中然后按顺序输入：

./configure


问题出现的原因：gmp和libgmp的版本不一致。

make(此时可能会出现错误)




解决的方法：更新一下源，选择阿里源即可。

通过：sudo gedit /etc/apt/sources.list


然后将网上找到的阿里源复制到文件中。输入 ：wq 保存并退出。

输入 apt-get update

apt-get upgrade


问题解决后，通过cd 进入ntl-11.3.2 的src文件中按顺序输入

./configure

make

make check

sudo make install


成功后

通过ls/usr/local/include/

出现 gmp.h NTL

通过 ls /usr/local/lib/


出现 libgmp.a libgmp.so libgmp.so.10.3.0 libgmp.la libgmp.so.10 libntl.a




说明安装成功了。
