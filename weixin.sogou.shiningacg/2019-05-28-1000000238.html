前言

当谷歌宣布参展GDC 2019（游戏开发者大会）时，许多人没有想到谷歌这位云计算巨头，推进高性能云游戏大规模商用的步伐如此之快。谷歌Stadia云游戏服务引起了广泛讨论，不过对手微软去年年底就正式宣布了Project xCloud，并宣布在19年面向公众测试，GPU厂商英伟达也早早提供了GeForce Now服务，以及索尼也提供了PlayStation Now（用来游玩PS3游戏，PS4直接兼容PS3比较困难）。巨头们已经准备很久了，借着基础技术进步之风，成熟的云游戏大规模商用正在到来。

 

云游戏试图解决一个终极问题，性能与便携共存——玩家希望游玩的设备可以是更便携的（嫌屏幕小的可以外接显示器，或者买折叠屏手机鸭），又能提供强劲的性能表现，而云游戏能同时满足这些，顺便附赠了即开即玩的爽快感——当然需要消耗些别的东西，比如高性能的网络。

云游戏本质上是在云端的虚拟机或者特殊容器上运行游戏，画面传到本地，同时将本地的操作传至云端。打个比方，过去玩家们想要游玩大作，需要”自己打一口井”，现在就只是向“自来水厂”订阅服务，然后按需付费——不过得先把自来水管扩宽。

云游戏实现的阻碍及其克服方法

贫穷的笔者在2014年时，蹭着亲戚家超快的网络，用自己孱弱的笔记本电脑，体验了英伟达刚公测的GRID云桌面，即在云端提供一个桌面系统给用户操作。当时英伟达给每个试用账户提供了24小时使用时间，每个云桌面系统分配了15GB内存、8个逻辑核心的虚拟CPU、性能接近于GTX680显卡的Tesla计算卡。同时英伟达提供了几个计算密集型应用的演示，比如网页上的流体模拟。在我的小破本上运行起了高性能图形应用，看起来真的很酷，深深震撼了我幼小的心灵（）。

但是受限于当时的技术，当画面动态元素较多时，这些元素就会比较模糊，或者画面卡顿，操作延迟也比较高（也与最近节点在东京和新加坡有关），所以打游戏是别想了……作为工作用的虚拟工作站，工程制图、3D建模、修个图什么的完全能满足的。

// 笔者自己购置的云服务器，无vGPU，只有1Mbps上行带宽，好在画面动态元素少，卡顿可以接受。

不过大厂们为什么要舍近求远搞个虚拟的云桌面呢，可以在此简单地提几点优势，包括但不限于节约计算资源（不是每个用户和业务同时满载的，虚拟化可以提升资源利用率）、控制关键项目文件的访问、便于移动办公等等。现在在某些行业云桌面已经是比较普及了，这些云桌面可能是运行在公司的私有云，或者购买的公有云服务上。

// 图片来自阿里云云工作站

实际上目前微软谷歌等大厂的云游戏方案并不是简单地在云桌面上跑游戏就完了，有许多办法来对云游戏这种特殊业务针对性优化。以下探讨一些云游戏实现需要应对的问题和一些应对方案。

画质——开源节流

开源

谷歌在GDC 2019上公布Stadia时，表示其目前能提供4K每秒60帧的游戏图像，未来终极目标是8K 每秒120帧。当然光看帧率和分辨率肯定是不行的，B站1080p 60fps的视频码率（视频压缩编码后每秒的数据量）上限只有6Mbps，而实际画质显然无法满足要求。谷歌要求是1080p至少25Mbps的带宽。

 

改善云游戏画质最直接的方法还是提升带宽，也就是“开源”，包括云游戏服务商的上行带宽和你自己的网络带宽（当然也有稳定性，延迟稳定、抗干扰、少丢包，像倒霉校园网就不满足这个……）。

节流

画质提升也不只依赖于网络的进步，新的视频编码算法也大有裨益。例如Google的AV1和较为常见的的H.265/HEVC编码，相比旧的常见编码方式（主要对比H.264）相同画质下视频码率减小约50% 。

// 更高级的编码方式会根据画面不同区域的复杂度，划分不同大小和数量的宏块进行编码。

可以预见，未来随着算力的提升，云和本地端可以高效运行更复杂压缩率更高的视频编解码算法，甚至于借助终端的AI（类似英伟达DLSS深度学习超采样来提升抗锯齿效能的思路），从节流的角度提升云游戏画质。

延迟受不了！

是的，前面提到笔者体验英伟达的早期云桌面时，因为种种问题，延迟较高游戏体验不佳。厂商也做了很多努力，现在巨头们的云游戏并不是直接在云桌面上跑游戏，像谷歌和微软深度优化了数据中心的软硬件和中间件来降低延迟。当然这也导致Stadia上游戏需要专门移植。

// 至于国内一些Windows平台的小型云游戏服务的配置和优化，参见文末参考资料4。笔者之前也体验过这类，个人认为体验一般。

 

同时谷歌微软等巨头还有个大杀器——遍地的数据中心，直接缩小物理距离和减少所经过的中间节点……所以云游戏是巨头的烧钱游戏。腾讯Start云游戏为了保证体验，目前也只在广东和上海开放测试。

 

经过一番骚操作，云游戏延迟表现如何呢？相比普通PC直接游玩，操作到显示的延迟增加了约70ms，整体尚在接受范围。

// 数据和制表来自DigitalFoundry。其中Project Stream是先前Google在Chrome浏览器提供的云游戏《刺客信条 奥德赛》体验活动。更详细说明参见参考资料2。

 

至于箭在弦上的5G，可以说5G是云游戏的重要推动者，在先行商用5G的Sub-6GHz频段下，已经可以实现小于20ms的通信延迟和千兆级带宽。后期5G毫米波频段商用后，性能会进一步提升。资费嘛……觉得贵是你的问题（）

图：OPPO Reno 5G版 和 5G网络下试玩云游戏版《王者荣耀》

 

当然从长期来看，一些硬伤使得云游戏延迟还是远远无法和本地设备媲美，所以游戏PC和主机是无法被替代的，尤其是对延迟敏感的竞技游戏。大多数RPG游戏的体验应该尚可，至少那群老外用Chrome浏览器先行体验云《刺客信条》表示还挺香的……

英伟达总裁黄仁勋：“…………如果你的问题是串流需要多长时间才能像一台PC那样好，我的回答是永远不可能。原因是有一个我们不知道如何解决的问题，那就是延迟。当你玩电竞游戏时，你需要在几毫秒内做出反应，而不是几百毫秒。这是一个根本性问题，也是物理法则。”

 

游戏阵容呢？

这才是最为玩家所关心的啊！而对于云游戏平台，开发者可能需要针对性移植——至少谷歌 Stadia是需要的，因为采用了定制的软件架构。微软表示开发者无需针对云游戏进行适配，延迟等问题由微软来处理。

对此微软还认真吐槽了一下谷歌Stadia云游戏：

Mike Nichols（Xbox首席营销官）：Emerging competitors like Google have a cloud infrastructure, a community with YouTube, but they don't have the content... You won't necessarily need a device over time, but you'll get the best experience with local processing power.

云游戏可能带来的其他的改变

交互上的创新提得比较多的就是，主播正玩着然后卡关了，观众不停吐槽该如何如何玩，焦躁的主播可以真的让观众“你行你上”，将控制权交给观众了。当然具体实现起来有不少困难。

同时游戏直播或许能有新的改观，观众可以直接看到云游戏传输来的画面，而非由主播的机器录屏编码推流，因此游戏直播门槛能进一步降低，尤其是性能有限的移动端设备。当然这些是笔者的不成熟想法，还请多指教。

番外：云计算大佬微软的特别尝试

微软曾经在2014年Build开发者大会上，展示了使用自家Azure云计算服务，来辅助Xbox One进行物体破坏的物理计算的操作。本地运算破坏场景只有2fps，而使用云端辅助物理计算则稳在32fps。这也是借用云计算提升游戏表现的好思路，对网络的延迟和带宽要求也比较低。

后来2018年上市的《除暴战警3》集成了该特性，可惜实际上市后被指缩水，反响不佳。

尾声

笔者作为一个轻度玩家，个人是很期待这类云游戏服务的普及的，既能按用量付费节约成本，避免初期对于设备和游戏的一次性大额投资，又能快速体验到大作，还不需要等待下载。不过对于重度玩家而言，他们所追求的极致的画面、性能和精准的操作，云游戏很长一点时间内还无法提供。

 

从游戏厂商的角度看，索尼选择与云计算和云游戏积淀深厚的老对手微软，在未来娱乐产品的云解决方案等方面合作研究，也能看出索尼感受到的压力和决心。

 

同时公有云第一巨头亚马逊在17年也收购了云游戏后端厂商GameSpark。可能这也是索尼压力的重要来源。而国内的互联网巨头里，腾讯在低调测试Start云游戏服务，期待其表现。

图：索尼总裁 吉田宪一郎 和 微软CEO 萨提亚·纳德拉

 

说不定过不了多久，大家都会是“云玩家”了 ：）
