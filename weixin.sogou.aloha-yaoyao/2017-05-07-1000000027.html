让团队保持优秀，远离60分标准的人和事

原创 2017-05-07 陆遥 了不起的陆茨比 了不起的陆茨比
了不起的陆茨比

aloha-yaoyao

遥遥的原创、译作和沉思录

我是如何产生这个想法的？


这一节讲了一些技术工作的背景，不感兴趣的同学可以直接跳到下一节，不影响阅读。


最近接手一个由纯 Python 实现的 BI 系统，主体是一个基于 Flask 做的 Web 前台，后面跑了一堆定时脚本出数据，另外加上一些简单的交互。由于以前需求排期排得紧，加上项目转手多人，工程实现把关不严，导致该系统代码质量一直饱受诟病；前期负责这一块的老员工在与我离职面谈中，也提到大家互相效仿随意写代码的做法、不思改进的「破窗效应」。结果是后人相比于前人，编码质量越来越差。


在总结观察、听取想法后，我觉得问题已经相当严重：Python 在大家的心目中一直都是脚本语言的形象，如果不能树立一个高标准，而是不问质量、随心所欲地编码，将进一步加深团队对 Python 的偏见（相对于正统工业级的 Java），让大家对这个系统失去信心，甚至会导致团队成员的进一步流失。因此重建标准、重朔信心，势在必行。我选择了一个最容易的点切入，就是编码标准。


其实 Python 在工业界里已经把这点做得非常好了，就是以 PEP8 为主的一系列标准。德鲁克说过，如果绩效不能被衡量，就没办法提高。基于此观点，我用 Pylint 封装了一套质检工具，可以在包维度给工程源码打分，然后通过不断重构优化，提高这个分数。如果低于某个合格线分数，则要强制要求改进。


这时自然产生了一个问题，如果满分是10分，多少分才算合格？


我一开始是想定6分的。后来改成了7分。在百分制中，60分默认是一个及格分数，但是对一个负责的职场人是不够的，更不能满足一个优秀团队的标准。70分才算是及格的起点。下面是我的分析和理由。


为什么60分标准是一个不可接受的标准？


60分是一个很微妙的数字，对于学生成绩而言是及格，对于员工绩效而言是达到期望，对于系统而言是可以工作workable。但不管是人还是事，都是处于一个动态的平衡过程中，受外部环境和时间原因的影响，没人能把一件事一直保持在60分，总有起起落落的波动。稍微不注意，波动就导致不及格。


对于职场上的人，波动表现在这个人一到办公室就应急，整天到处扑火，时刻胆战心惊，精力只够应急式工作；而稍有空闲，却小富即安，懒于推演未来局势、懒于建立问题统筹框架，所以一直处于被动挨打状态。


对于职场上的事，波动表现在系统稍不管就又挂了、数据稍不盯就又跌了、业务稍不催就没人理了、进度稍不跟就一直卡在那了。或者就是前一节提到的破窗效应，每件事做到上一件标准的及格线，几轮下去，就彻底恶化了。


60分标准的人，干着60分标准的事，如果不再负责当前的业务，表现出来的态度就是不再认可自己曾经的工作。因为60分标准带来的波动，给他们造成了太多痛苦的回忆。出于个人名誉或者摆脱后续问题麻烦的需要，他们也急于和以前60分的成果撇清关系。


既然如此，为什么还有那么多人能忍受60分波动的折磨而生存？有句俗语叫只见贼挨打，不见贼吃肉。如果能忍受种种问题、冒着风险行事，则回报也是优厚的：花最少的精力，做刚刚好的事情，拿到想要的东西，然后转移阵地。听起来很合理，毕竟追求了最高的投资回报比。但是对于团队来说，没有任何一件事情是一次性交易，如果有负面影响，会一直持续。


60分标准的人和事，会给团队产生哪些伤害？


60分标准人和事遗留的问题，需要留下的团队成员花费额外的精力去收拾。举个例子，如果有人要借技术债，不是不可以，但是冤有头债有主，我一直会盯着你本人把债还掉。如果我觉得你在任期内还不了债，我会拒绝你的借贷要求。对于每一位团队管理者，都在做风投管理和银行业务。60分标准的人和事，是劣质客户，要拒之门外。


60分标准人和事的做事风格，会损害到团队其他积极向前看、有长远发展眼光的成员积极性。如果让60分标准的人和事在你的团队投机成功，会挑战团队管理者的威信和说服力。如果每个人都追求短平快出结果，没有人对长远目标负责，团队的持续发展难以保证。不要等到所有的短期的投资项目做完，才开始考虑长期投资项目。


如果有人要问，多少分才是合格的标准，我的答案是起码70分。70分可以抵御一些微小的波动，不会让工作随时都有可能变得很难看；70分相对60分也不需要花费太多的额外精力，却能减少投机者的杠杆效应，让追求良性快速成果的成员多一分思考；让愿意做长远贡献的成员多一分安定。


70分算是给团队交了一份不太贵的投名状，表明你不会贪心捞一把就走，你还是自己人，还会在未来为自己曾经的工作而骄傲，而不是逃避。


究竟多少分才真正适合你的团队？


70分是一个底线。当工作成果状态低于70分，警报会响起，你还有缓冲时期去修复它们，而不是直接落到老板和客户面前丢人。这种丢人不是在家庭聚会上把歌唱跑调，而是会直接不可逆转地对你的名誉产生伤害。久而久之，大家会失去你的信任。如果你处在关键位置，甚至有人会因为你而想「换个环境」。


70分标准也会帮助过滤一些看起来很紧急但是不重要的事情，避免不必要的资源损失。毕竟，ATM机还有大额取款提醒，没有那么多的急事非得逼迫我们降低标准。这些事情大多数也都没什么价值。


对于我而言，80分是一个最佳的标准。根据二八原则，80分已经足以在投资和回报之间取得完美平衡。我一直以80分的标准要求我的团队，当然也以同样的标准要求我自己。


如果团队中有人落后了，该怎么办？如果不愿意接受提高标准，则需要适当调整人员配置，优秀的团队应该对60分标准零容忍。最好的结局是大家一起制定一个不低于70分的标准，然后一起去尝试改进。


团队成员之间要达成一致的期望，要避免一群高分学霸和只有及格要求的另一群人在一起待太久，否则双方都有可能流失。毕竟，团队健康成长的最佳标准之一，就是互相满意。
