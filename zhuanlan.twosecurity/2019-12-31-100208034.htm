一个汇聚网络安全资源的渗透测试框架。

Lockdoor（锁门？）

名字来源于作者（Sofiane Hamlaoui）和他的朋友（Rafik）的突发奇想。

为什么创造？

在 Lockdoor 出现之前，作者有一个用来在 github 上下载 Public 的 shell 脚本，方便自定义、添加、删除等等。

突然有一天，他灵光一现，为什么不做一个共享的框架，这样就可以了解学习更多自己不知道的新工具。

于时，Lockdoor 诞生了。

Lockdoor

一个旨在帮助进行渗透测试的框架。

为渗透测试人员设计的 DIY 目录：/pentest/

与众不同

现在市面上那么多形形色色的渗透测试框架，他们都有那么多类似的地方，为什么还要用 Lockdoor ？

这也是作者创作的初衷所在，作者说，Lockdoor 的有其独特的附加值，它最好的地方是添加了测试人员最喜欢和最常用的工具、自动化测试流程以帮助你更快地完成工作！工具+资源，这就是使 Lockdoor 成为一种新工具的原因。

  • 包含测试人员最喜欢和最常用的工具。
  • github 上一些大佬制作的工具，或一些黑客团队的私人工具。
  • 自定义化，轻松添加、删除工具
  • 资源，Pentesing 和 Security Assessment 结果报告模板，Pentesting 演练示例和模板等。
  • 备忘单，顾名思义，什么东西都会有遗忘的时候。

搭建

支持的环境：

Kali Linux、Ubuntu、Arch Linux、Fedora、Opensuse、Windows (Cygwin)

docker 一键搭建（环境：Win 10）

docker hub：https://hub.docker.com/r/sofianehamlaoui/lockdoor

拉取 docker 镜像（需要一点时间，将近 1G）：

docker pull sofianehamlaoui/lockdoor

运行新的容器：（避坑，作者给出的这一步很大几率会出问题！）

docker run -it --name lockdoor-container -w /home/Lockdoor-Framework --net=host sofianehamlaoui/lockdoor
目录为空！原因不明，可能是运行时命令错乱导致 Lockdoor 主文件未导入到 linux image 下的目录中。

❗使用以下命令运行：

docker run -it sofianehamlaoui/lockdoor

停止容器：

docker stop （容器ID）

重新运行已停止的容器：

docker start -i sofianehamlaoui/lockdoor

先开启一个容器，然后启动：

./install.sh

进入安装步骤。（中途会询问安装路径，默认【Enter】即可）

./lockdoor

可能会提示需要网络才能运行，那就配网吧。
