原文

每年Rust社区会聚在一起公布roadmap. 今年， 除了调查之外，我们还在12月份进行呼吁社区博客帖子，结果是在几周内大家撰写100篇博客文章。结束后，结果合并成　2018 roadmap RFC.

Rust: 2018 发行版

今年我们将交付 Rust 2018, 　这将是从1.0之后的第一个主要的发行版。

像往常一样，我们继续每六周发布。 但是我们将在今年三个版本（Rust 1.29 - 1.31）中选定一个作为Rust 2018　这个新的Rust“发行版”将是整年功能稳定的顶点，同时这些特性将带有精美文档，工具和库。

发行版的意图是标记Rust进化中的主要里程碑，它收集了一些新的特征或习惯用法，整体上改变了使用Rust的经验。 隔几年，来评估我们以六周为周期的交付工作。 讲述RUST发展的更大的故事。 将整个我们整体的工作打造成闪光的产品发布。

我们期望每一个发行版将有一个核心专注主题。　考虑到Rust 1.0 作为　“Rust 2015” ，　那我们将有：

  • Rust 2015年：稳定性
  • Rust 2018年：生产力

哪些会成为Rust 2018?

Roadmap 并不是说确定那些会在 Rust 2018，但我们有好的想法，我们将会覆盖以下主要想法。

文档改进

Rust 2018 release 的目标之一，对于新增和改进的特性和惯用法提供高质量的文档。　《The Rust Programming Language book》 在过去的18月里，已经完成了重写，一年内随着特性到达稳定的编译器支持，它也将更新。 Rust By Example 今年也将进行修改。还有许多第三方的书籍，如Programming Rust ，会将印刷出版。

语言改进

2017’s ergonomics initiative 队列中最为优先的语言工作,　几乎所有被接受的RFCs今天都在nightly分支上提供，在接下来的几个月里，它将被打磨和稳定。这些中生产力的改进，会做作为“头条”在主干发布

  • Ownership 系统改进,（所有权系统改进），包括可以通用“"non-lexical lifetimes””来实现更灵活的标记借用,改进的模式匹配集成.
  • Trait 系统改进，包括期待已久的处理类型抽象的 impl Trait语法
  • 模块系统改进　聚焦在增加清晰性和降低复杂度
  • Generators/async/await：在最优先的异步编程支持方面，工作在快速的进展。

另外，　我们期望有更多一些主要的特性在Rust 2018 发布，能稳定起来，包括SIMD, custom allocators 和 macros 2.0

编译器改进

Rust 1.24 增量再编译已经支持，在稳定版的编译器已是默认打开，该特性已经使重构建比新构建的速度快得多，但在这一年的时间里，我们预计fresh builds 和re-builds 都会得到持续的改进。 在Rust 2018, 编译器性不能成为生产效率的主要的障碍。

工具改进

Rust 2018 将看到高质量的 RLS-1.0 (Rust Language Server)　和rustfmt 我们将继续改进。

我们将继续改进 Cargo 稳定客户注册、公共依赖和改进Profile system. 我们也期望将来在Cargo build system integration, Xargo integration, 和 custom test frameworks　工作，

尽管目前还不清楚这其中有多少将在Rust 2018之前完成。


Library 改进

在我们去年的工作之上 　我们将发布1.0的　Rust API guidelines book, 继续推动重要的libraries 达到 1.0 状态， 通过改进cookbook 的努力来提高发现能力，并在特定领域的libraries 进行大量投入 - 我们将在下面看到。


WebSite 改进

作为Rust 2018的一部分，我们将彻底检修Rust 网站，使它对CTOs和工程师都很有用。在您的用例下，应该更容易找到信息来帮助评估Rust ，并保持最新的工具和生态系统的改进。

四个目标领域

Rust 2018 目标之一是演示在特定的领域使用Rust的生产力。　今年我们突出选了四个领域进行投入：

  • 网络服务 Rust的可靠性和低的资源占用非常匹配网络服务和基础设施，特别在高伸缩领域。
  • Command-line apps (CLI).　Rust的可移植性、可靠性、工程效率 以及产生静态二进制文件，综合这些能力对编写CLI应用程序产生巨大影响。
  • WebAssembly. 　“wasm” web 标准允许加载本地二进制到主流的浏览器，但是GC支持还要多年. 　这个领域非常符合Rust的定位，为JS程序员提供一个合适的切入点。
  • Embedded devices. Rust　有潜力使资源受限的设备编程更加高效和有趣!　　我们希望Rust在嵌入式编程今年达到一流的地位。

今年每一个领域都有一个专门工作小组。这些工作组将以交叉的方式工作，与语言、工具、库和文档工作相结合。
